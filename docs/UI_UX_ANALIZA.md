# UI/UX анализа репозиторијума (scruma-web)

## Општа оцена
Сајт има добру основу (конзистентан визуелни стил, јасна хијерархија секција, fallback стања за API), али постоји простор за унапређење приступачности, информационе архитектуре и мобилне употребљивости.

## Кључни налази

### 1) Приступачност је ограничена на мобилним уређајима
- У `layout.tsx` је подешен viewport са `maximum-scale=1` и `user-scalable=no`, што онемогућава pinch-to-zoom за кориснике којима је увећање неопходно.
- Предлог: уклонити ограничење зума и оставити стандардни responsive viewport.

### 2) Главна навигација је претрпана (desktop)
- Header има велики број ставки у примарном менију (9 линкова), што повећава когнитивно оптерећење.
- Предлог: груписати „Велика сала / Сале / Куглана / Теретана / Базен“ у један dropdown (нпр. „Садржаји“), или применити мегамени.

### 3) Контраст и читљивост навигације зависе од позадине
- На иницијалном стању header-а линкови су бели са сенком, док је позадина транспарентна, што на светлим деловима hero слике може умањити контраст.
- Предлог: додати јачи gradient испод навигације, минимални contrast guard (полупрозирна трака), и јасне `:focus-visible` стилове.

### 4) Недостају „брзе радње“ у hero секцији
- `Hero` подржава CTA дугмад, али се на почетној шаље празан низ (`ctas={[]}`), па корисник нема јасан први корак.
- Предлог: додати 2 CTA акције: „Погледај садржаје“ и „Контакт / Радно време“.

### 5) Footer делује као листа текста, не као навигација
- У footer-у су ставке приказане као `<li>` без линкова.
- Предлог: претворити ставке у кликабилне линкове ка релевантним страницама, додати контакт блок и брзе информације (радно време, адреса, телефон).

### 6) Мапа на почетној вероватно није функционална
- `iframe` користи `src="https://www.google.com/maps/embed?pb=..."` (placeholder), што указује да embed није финално подешен.
- Предлог: унети валидан embed URL и fallback линк „Отвори у Google Maps“.

### 7) Тренутни UX при API грешци може да прекине ток
- Грешке су јасно приказане (добро), али када API није доступан корисник добија много „празних“ стања.
- Предлог: додати информативне fallback модуле по секцијама (нпр. „Нема тренутно објава“) са линком ка контакту/обавештењима.

### 8) Недостају микро-интеракције приступачности
- Нема јасно дефинисаних глобалних `:focus-visible` стилова за тастатурну навигацију.
- Предлог: додати глобални стил фокуса за линкове, дугмад и контроле у header-у/мобилном менију.

### 9) Садржајна стратегија је добра, али треба боље UX сигнализирати „празан CMS“
- Пројекат правилно приказује статус „Садржај није унет у админ панел“, али поруке могу бити више кориснички оријентисане.
- Предлог: уместо техничке формулације, додати кратко објашњење и очекивано време ажурирања („Садржај ће бити ускоро објављен“).

## Приоритетни план (30 дана)

### Брзе победе (1–3 дана)
1. Уклонити `user-scalable=no` и `maximum-scale=1`.
2. Додати глобалне `:focus-visible` стилове.
3. Попунити hero CTA дугмад на почетној.
4. Исправити Google Maps embed + fallback линк.

### Средњи приоритет (4–10 дана)
1. Рефактор навигације (груписање ставки).
2. Footer конверзија у праву секундарну навигацију.
3. Уједначити празна/грешка стања са корисничким порукама.

### Већи захват (10+ дана)
1. Дефинисати design tokens за контраст/focus/spacing по брекпоинтима.
2. Увести basic UX telemetry (кликови на CTA, depth scroll) ради верификације побољшања.

## Предложени KPI
- +15–25% CTR на примарни CTA на почетној.
- -20% bounce rate на мобилним уређајима.
- +10% корисника који отварају контакт страницу са почетне.
- 0 критичних приступачносних проблема за zoom/focus/navigation.
